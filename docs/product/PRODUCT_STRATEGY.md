# Tengra Product Strategy

## 🎯 Target Audience
- **Solo Developers**: Individuals looking for a local-first, privacy-focused AI coding partner.
- **Indie Teams**: Small teams that need shared context without compromising data sovereignty.
- **Local-First Power Users**: Developers who prefer on-device compute (Ollama, Llama.cpp) over cloud-only solutions.

## 🚀 Core Value Proposition
Tengra is a unified AI assistant that bridges the gap between local LLMs and cloud providers, offering a secure, high-performance desktop environment for coding, research, and automation.

### Key Differentiators:
1. **Privacy First**: "Bring Your Own Key" (BYOK) model. All sensitive data remains local.
2. **Local AI Native**: First-class support for Ollama and Stable Diffusion (SD-CPP) with automatic cloud fallbacks.
3. **Multi-Model Collaboration**: Ability to engage multiple models (e.g., GPT-4 + Claude + Ollama) in a single session.
4. **Tool-Enabled Agents**: Extensible agent system that can interact with the filesystem, terminal, and browser.

## 🛑 Non-Goals (What Tengra is NOT)
- **Not a SaaS**: We do not host your data or models.
- **Not a Model Provider**: We provide the interface and orchestration, not the actual LLM compute (unless running locally).
- **Not a Full IDE Replacement**: Tengra complements your IDE (VS Code, Cursor) rather than replacing it.

## 🛣 Roadmap Summary
- **Phase 1 (Current)**: Solidify multi-model support, SSH workspaces, and local AI stability.
- **Phase 2**: Mature Automation Workflows and MCP plugin ecosystem.
- **Phase 3**: Enhanced hybrid GPU orchestration and collaborative memory.
