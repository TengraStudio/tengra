## Reporting a Vulnerability

We take the security of Orbit very seriously. If you discover a security vulnerability, please do NOT open an issue.

Instead, please email [orbit@tengra.studio](mailto:orbit@tengra.studio) with a detailed description of the vulnerability.
We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress.
We will handle all security vulnerability reports with the priority they deserve.

### What to include in your report

*   The type of vulnerability (e.g. buffer overflow, SQL injection, cross-site scripting).
*   Full paths to source files where the vulnerability can be found.
*   The location of the affected source code (tag/branch/commit or direct URL).
*   Any special configuration required to reproduce the issue.
*   Step-by-step instructions to reproduce the issue.
*   Proof-of-concept or exploit code (if possible).
*   Impact of the vulnerability, including how an attacker could exploit the issue.

### Disclosure Policy

*   We ask that you do not disclose the issue to the public for at least 90 days after reporting it to us, or until we have released a fix.
*   We will provide you with a credit for the discovery in our release notes and on our website, if you wish.
